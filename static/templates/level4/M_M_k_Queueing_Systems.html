Introduction

The M/M/k queue extends the M/M/1 model to k identical servers. Customers arrive according to a Poisson process and are served by any available server with exponentially distributed service times. This model is widely used to analyze call centers, bank tellers, and other multi-server systems.

Key Concepts & Formulas

  • Model Assumptions:
    • Poisson arrivals with rate λ
    • Exponential service times with rate μ per server
    • k identical servers
    • Infinite waiting room
    • FCFS discipline
  • Traffic Intensity:
    \[ \rho = \frac{\lambda}{k\mu} \]
  • Stability Condition: \(\rho < 1\)
  • Stationary Distribution:
    \[ \pi_n = \begin{cases} \frac{(\lambda/\mu)^n}{n!}\pi_0 & \text{for } 0 \leq n \leq k \\ \frac{(\lambda/\mu)^n}{k!k^{n-k}}\pi_0 & \text{for } n > k \end{cases} \]
  • Probability of Waiting: (Erlang C formula)
    \[ C(k,\lambda/\mu) = \frac{(\lambda/\mu)^k}{k!(1-\rho)} \cdot \frac{1}{\sum_{i=0}^{k-1} \frac{(\lambda/\mu)^i}{i!} + \frac{(\lambda/\mu)^k}{k!(1-\rho)}} \]

Worked Example

Consider an M/M/2 queue with λ = 6 customers/hour and μ = 4 customers/hour per server.

Traffic intensity: \(\rho = \frac{6}{2 \cdot 4} = 0.75\)

For n = 0: \(\pi_0 = \frac{1}{1 + \frac{6}{4} + \frac{(6/4)^2}{2!(1-0.75)}} = \frac{1}{1 + 1.5 + 4.5} = 0.143\)

For n = 1: \(\pi_1 = \frac{6/4}{1} \cdot 0.143 = 0.214\)

For n = 2: \(\pi_2 = \frac{(6/4)^2}{2} \cdot 0.143 = 0.161\)

For n > 2: \(\pi_n = \frac{(6/4)^n}{2 \cdot 2^{n-2}} \cdot 0.143 = 0.143 \cdot 0.75^n\)

Practice Problems

Problem 1

What is the key difference between M/M/1 and M/M/k queues?

M/M/1 has a single server, while M/M/k has k identical servers. This affects the service rate (kμ vs μ) and the probability distribution of the number of customers in the system.



Problem 2

For an M/M/3 queue with λ = 9 and μ = 4, what is the traffic intensity?

\(\rho = \frac{\lambda}{k\mu} = \frac{9}{3 \cdot 4} = \frac{9}{12} = 0.75\)



Problem 3

What is the Erlang C formula used for?

The Erlang C formula gives the probability that an arriving customer must wait (all servers are busy), which is crucial for capacity planning and service level analysis.
